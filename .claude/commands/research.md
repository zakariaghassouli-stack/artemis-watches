## Research Codebase

Lis en profondeur le dossier ou la feature indiqué(e). Documente :
1. **Structure** : Quels fichiers, comment ils sont organisés
2. **Patterns** : Quels design patterns sont utilisés
3. **Dépendances** : Quelles libs/composants sont impliqués
4. **Types** : Les interfaces TypeScript pertinentes
5. **État actuel** : Ce qui marche, ce qui est incomplet, les TODO

Écris UNIQUEMENT des faits observés. Pas de suggestions, pas d'opinions.
Output : `docs/research/$ARGUMENTS.md`
